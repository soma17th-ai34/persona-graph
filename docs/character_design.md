# PersonaGraph Character Design

## 디자인 방향

PersonaGraph의 캐릭터는 사람형 아바타가 아니라 손바닥 크기의 AI 오브제에 가깝다. 화면 얼굴 대신 빛, 움직임, 촉감, 말투로 성격을 전달한다.

핵심 기준:

- 작고 곁에 둘 수 있는 형태
- 무디스플레이 얼굴 또는 표정 라이트
- 기능보다 관계성이 먼저 느껴지는 캐릭터
- 발표용으로 설명 가능한 명확한 역할
- 피지컬 AI 프로젝트로 확장 가능한 외형 언어

## 캐릭터 세트

- 노리: 큰 아이디어를 첫 MVP 장면으로 줄이는 조율자
- 오르빗: 흐름과 실패 지점을 안정적인 궤도로 잡는 설계자
- 밀밀: 기능을 사용자가 느끼는 순간으로 바꾸는 감정 번역가
- 소리: 기술 결과를 기억에 남는 발표 장면으로 만드는 연출가
- 모리: 약한 가정과 숨은 리스크를 찾는 관찰자
- 결: 흩어진 의견을 하나의 실행 가능한 결론으로 엮는 종합가
- 자리: 아이디어를 검증 가능한 실험으로 놓는 실험 설계자
- 샐리코어: 현재의 목표, 깊이, 현실성을 함께 챙기는 기술 동료

## UI 적용

각 페르소나는 실행마다 캐릭터 풀에서 임의로 하나를 배정받는다. 저장된 실행 로그에는 배정된 캐릭터가 함께 저장되어, 다시 불러와도 같은 캐릭터가 표시된다.
