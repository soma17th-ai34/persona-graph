# PersonaGraph UI 개선 스펙

## 관련 문서

- [design_tokens.md](./design_tokens.md): ChatGPT/Gemini형 채팅 UI로 재구성하기 위한 디자인 토큰
- [tasks.md](./tasks.md): UI-only 가능 여부와 단계별 작업 계획

## 개요

PersonaGraph 의 UI를 개선한다.

## MainScreen

### 철학

chatGPT, Gemini 같은 기존 LLM 웹 서비스와 같은 UI를 그리는 것을 목표로 한다.

### 디자인 토큰

chatGPT의 디자인 시스템을 그대로 채용한다.

실제 구현에서는 특정 서비스의 비공개 디자인 토큰을 복제하지 않고, 중립적인 채팅 제품 UI 토큰으로 재해석한다.

### UI

1. 레프트 사이드바
- 접거나 필 수 있어야 한다.
- 채팅 내역을 보여줘야 한다.

2. 헤더
- 좌측에 personaGraph 라는 브랜드명만 남긴다.
- 우측에 기존에 있던 거 그대로 쓴다. + 실행 설정 넣기 누르면 모달 띄우기

3. 메인 페이지
- 채팅 내용이 없다면, 어떤 문제를 해결하고 싶으신가요? 텍스트를 중앙에 둔다. + 샘플 시나리오
- 그리고 그 바로 밑에 채팅 폼을 넣는다.
- 채팅을 입력하면, 이제 채팅창처럼 진행한다. 내가 입력한 거는 우측에, 페르소나들이 입력하는 거는 좌측에 나온다.
- 스크롤을 마지막 채팅 기준으로 내려보낸다.

### 가능 여부 파악

#### 원칙

오직 UI만 건들여서 할 수 있는지 파악한다.

#### 요소

- 페르소나가 생성되면, 생성될 때 안녕하세요 저는 누구고 어떤 도움을 드립니다를 채팅에 먼저 나오게 할 수 있나? 그러면 굳이 페르소나 상세 페이지를 보여줄 필요가 없을 거 같다.
- 그리고 답변 생성이 길어진다면, 현재 누가 의견을 내고 있다를 보여줄 수 있나?
- 그리고 페르소나마다 채팅 버블 디자인의 차이를 줄 수 있나? 테두리만 좀 바꿔주면 될듯
- 첫 문제를 정의하고 채팅 내역 보냈을 때 대화 구성을 설정할 수 있게 해줄 수 있나? ~~하고 싶어. 라고 채팅 보내면 페르소나 수, 이어말하기 라운드 등등 실행 설정에서 직접 해야 되는 거를 할 수 있게 채팅창 안에서 보여주면 좋겠어. 그리고 토론 때 마다 묻기, 디폴트값으로 설정하기 등등
- 그리고 예를 들어 페르소나 수라면 사용자가 이해할 수 있게 서브 텍스트도 넣어주기.
- 마지막 채팅을 기준으로 스크롤을 자동으로 아래로 내릴 수 있나?
